A - AtCoder Line

高桥站即将从 XXX 站前往 YYY 站，只需使用进站和出站列车中的一列。

求这趟列车在 ZZZ 站是否停车。

Solve

B - Typing

Problem

高桥尝试使用键盘输入由小写英文字母组成的字符串 SSS 。

他在键入时只看键盘，不看屏幕。

每当他错误地输入一个不同的小写英文字母时，他就立即按下退格键。然而，退格键被破坏了，因此误键入的字母没有被删除，实际键入的字符串是 TTT 。

他没有误按小写英文字母以外的任何键。

TTT 中没有被误输入的字符称为正确输入字符。

确定正确键入的字符在 TTT 中的位置。

Solve

双指针处理一下即可，iii指针扫错误字符串TTT,然后在扫的过程中如果和正确字符串SSS中的jjj指针所在位置的字符相同，jjj指针前移

C - Standing On The Shoulders

Problem

有 NNN 个巨人，他们的名字分别是 111 到 NNN 。当巨人 iii 站在地上时，他们的肩高是 AiA_iAi​ ，头高是 BiB_iBi​ 。

你可以选择 (1,2,…,N)(1, 2, \ldots, N)(1,2,…,N) 的排列 (P1,P2,…,PN)(P_1, P_2, \ldots, P_N)(P1​,P2​,…,PN​) ，并根据以下规则堆叠 NNN 个巨人：

  • 首先，将 P1P_1P1​ 巨人放在地上。巨人 P1P_1P1​ 的肩膀距离地面的高度为 AP1A_{P_1}AP1​​ ，头部距离地面的高度为 BP1B_{P_1}BP1​​ 。

  • 为了 i=1,2,…,N−1i = 1, 2, \ldots, N - 1i=1,2,…,N−1 的顺序，把巨人 Pi+1P_{i + 1}Pi+1​ 放在巨人 PiP_iPi​ 的肩膀上。如果巨人 PiP_iPi​ 的肩膀距离地面的高度是 ttt ，那么巨人 Pi+1P _{i + 1}Pi+1​ 的肩膀距离地面的高度就是 t+APi+1t + A_{P_{i + 1}}t+APi+1​​ ，他们的头距离地面的高度就是 t+BPi+1t + B_{P_{i + 1}}t+BPi+1​​ 。

求最上面的巨人 PNP_NPN​​ 的头部距离地面的最大可能高度。

Solve

最终的结果是nnn个巨人的肩高加上一个巨人头到肩的高度，所以直接贪心找出头到肩的高度最大的加上nnn个巨人的肩高即可

D - Permutation Subsequence

Problem

给你一个 (1,2,…,N)(1, 2, \dots, N)(1,2,…,N) 的排列组合 P=(P1,P2,…,PN)P = (P_1, P_2, \dots, P_N)P=(P1​,P2​,…,PN​) 。

如果一个索引序列 (i1,i2,…,iK)(i_1, i_2, \dots, i_K)(i1​,i2​,…,iK​) 同时满足以下两个条件，那么这个索引序列被称为好索引序列：

  • 1≤1\leq1≤i1i_1i1​ < i2i_2i2​ < …\ldots… < iki_kik​ ≤\leq≤ NNN .
  • 子序列 (Pi1,Pi2,…,PiK)(P_{i_1}, P_{i_2}, \dots, P_{i_K})(Pi1​​,Pi2​​,…,PiK​​) 可以通过重新排列一些连续的 KKK 整数而得到。
    形式上，存在一个整数 aaa ，使得 {\lbrace{Pi1,Pi2,…,PiKP_{i_1}, P_{i_2}, \dots, P_{i_K}Pi1​​,Pi2​​,…,PiK​​}=\rbrace=}= {\lbrace{ aaa , aaa + 111, …\dots… ,aaa + KKK - 111}\rbrace}

求所有好的索引序列中 iK−i1i_K - i_1iK​−i1​​ 的最小值。可以证明，在此问题的约束条件下，至少存在一个好的索引序列。

Solve

仔细读题可以发现其实就是让我们找一个有最长连续数字的长度为kkk的区间，因为PPP 是111~NNN的排列组合，所以数字iii所在的位置就是PiP_iPi​ ，我们用一个数组将每个数字的位置记录下来，然后遍历长度为kkk的区间，然后找到 iii 到 (i+k−1)(i + k - 1)(i+k−1) 中差距最远的两个位置，及最大值和最小值，暴力会超时，因为找区间最值所以可以用ststst表。有因为是固定区间所以我们还可以用滑动窗口来解决。

E - Clique Connect

Problem

给你一个加权无向图 GGG ，有 NNN 个顶点，编号为 111 至 NNN 。最初， GGG 没有边。

您将执行 MMM 次操作来为 GGG 添加边。 iii -th 操作 (1≤i≤M)(1 \leq i \leq M)(1≤i≤M) 如下：

  • 给你一个由 KiK_iKi​ 个顶点组成的顶点子集 Si={Ai,1,Ai,2,…,Ai,Ki}S_i=\lbrace A_{i,1},A_{i,2},\dots,A_{i,K_i}\rbraceSi​={Ai,1​,Ai,2​,…,Ai,Ki​​} 。对于每一对 u,vu, vu,v ，即 u,v∈Siu, v \in S_iu,v∈Si​ 和 u<vu < vu<v ，在顶点 uuu 和 vvv 之间添加一条边，权重为 CiC_iCi​ 。

完成所有 MMM 操作后，确定 GGG 是否相连。如果是，求 GGG 最小生成树中各条边的总权重。

Solve

本题也有两个解法，解法一是加虚点，就是对集合SiS_iSi​加一个虚点，让这个集合所有的点都向它连接一条权重为CiC_iCi​，这样集合中所有点就都相连了，最后只要减去到虚点的权重及可，这个我们可以在开始刚添加虚点时就减去。

解法二是根据KruskalKruskalKruskal 算法的性质来实现的，因为KruskalKruskalKruskal 算法是找最短的边，所以我们可以直接根据CiC_iCi​ 从小到大的顺序来形成最小生成树。

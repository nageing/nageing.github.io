I. Cyclic Apple Strings

Problem

给定一个长度为 nnn 的字符串 sss ，它只由 0 和 1 组成。我可以任意多次执行以下操作：选择 sss 和一个正数 kkk 的子串，然后循环地将子串 kkk 向左移动位置。

字符串 aaa 是字符串 bbb 的子串，当且仅当 aaa 可以通过删除 bbb 开头和结尾的一些字符（这些字符可以是全部，也可以是其中的任何一个字符）来获得时， aaa 才是字符串 bbb 的子串。(可以是一个字符，也可以是全部字符）。

假设有一个字符串 s=s0s1s2…sm−2sm−1s=s_0s_1s_2\dots s_{m-2}s_{m-1}s=s0​s1​s2​…sm−2​sm−1​ ，将它向左移动 kkk 位将得到 sk′sk′+1…sm−1s0s1…sk′−2sk′−1s_{k'}s_{k'+1}\dots s_{m-1}s_{0}s_{1}\dots s_{k'-2}s_{k'-1}sk′​sk′+1​…sm−1​s0​s1​…sk′−2​sk′−1​ ，其中 k′=k mod mk'=k\bmod mk′=kmodm 。

当且仅当 1≤i<n1 \leq i < n1≤i<n 、 si≤si+1s_i \leq s_{i+1}si​≤si+1​ 都是0和1时，仅由0和1组成的字符串 sss 被排序。在这里，字符是根据其数值进行比较的。

例如，将字符串 0101100 的子串 1011 向左移动两个位置，结果是 0111000。

我需要找出使 sss 排序所需的最少操作数。

Solve

当111与111之间有000时，我们就操作一次把111全部后移，最后如果结尾是000那我们就再移动一次（注意，全是000时不用移动）

Problem

小指头发明了一个双人游戏，我们轮流向她挑战。游戏具体如下

有 nnn 个整数 1,2,3,…,n1,2,3,\dots,n1,2,3,…,n 从左到右**排成一行。我和小指头轮流尝试以下操作：

  • 如果剩余整数的位XOR和不是 000 ，则在不改变剩余数字顺序的情况下，从该行中移除最左边或最右边的一个整数。

如果当前玩家无法移动，则输掉游戏。

假设 "我 "每轮总是先开始，而我们双方都希望通过下最佳棋步来取得胜利，那么这盘棋共下 TTT 轮。问题是 "我 "能否在每一轮对局中取得胜利。

多个数字 a1,a2,…,ama_1,a_2,\dots,a_ma1​,a2​,…,am​ 的比特 XOR 和用 a1⊕a2⊕⋯⊕ama_1 \oplus a_2 \oplus \dots \oplus a_ma1​⊕a2​⊕⋯⊕am​ 表示。尤其是空集的 XOR 和为 000 。

XOR 运算用 ⊕\oplus⊕ 表示，是一种逐位比较两个二进制数的二进制运算。在每个位置上，如果对应的位不全是 111 或不全是 000 ，结果就是 111 ；否则，就是 000 。

通过找规律可以发现，当 (n%4==0∣∣n%4==1n\%4 == 0 || n\%4 == 1n%4==0∣∣n%4==1 )时“我”一定会赢，其他情况则对方赢

B. Countless Me

Problem

给定一个长度为 nnn 的非负整数数组 aia_iai​ ，我可以执行以下操作不超过 nnn 次：

  • 选择 i,ji,ji,j ，使得 1≤i,j≤n1\le i,j \le n1≤i,j≤n ， 0≤x≤1090\le x\le 10^90≤x≤109 ，然后设置 ai←ai+xa_i\leftarrow a_i+xai​←ai​+x 和 aj←aj−xa_j \leftarrow a_j-xaj​←aj​−x 。

我必须确保每次操作后所有 aia_iai​ 都保持非负整数，由我们中的一人提醒。

我的目标是使运算后 a1 ∣ a2 ∣ … ∣ ana_1 \ | \ a_2 \ | \ \ldots \ | \ a_na1​ ∣ a2​ ∣ … ∣ an​ 的值最小。这里， ∣|∣ 表示位操作 OR 运算符， x ∣ yx \ | \ yx ∣ y 表示两个非负整数 xxx 和 yyy 的位操作 OR。

位顺向 OR 运算逐位比较两个二进制数。在每个位上，如果至少有一个位是 111 ，那么结果就是 111 ；否则，就是 000 。例如，二进制数 10=1010210 = 1010_210=10102​ 和 12=1100212 = 1100_212=11002​ 的逐位 OR 结果是 14=1110214 = 1110_214=11102​ ，而 3=001123 = 0011_23=00112​ 和 5=010125 = 0101_25=01012​ 的逐位 OR 结果是 7=011127 = 0111_27=01112​ 。

Solve

nnn次操作一定可以让数组最后变成我们想要的样子，所以我们只需要让它或运算之后最小即可。先累加数得到sumsumsum，然从高到低判断二进制位，如果当前位可以为000（及当前位为000最后也可以将sumsumsum 分配完），如果一定要取得话我们再判断一下剩下的sumsumsum 能不能分配nnn个（nnn个数），如果不能则看最多可以分配多少。

Problem

在一张长桌上， nnn 个座位排成一排，从左到右 iii 个座位上摆放着分量为 aia_iai​ 的菜肴。由由子从 sss 个座位开始，每秒结束时，她可以移动到与当前座位相邻的任何座位（她也可以选择留在当前座位）。她所到达的座位上的任何菜肴都会被她吃掉。

由由子想知道她能吃的菜肴的最大总份量。对于满足 1≤s≤n,1≤t≤2n1 \le s \le n, 1 \le t \le 2n1≤s≤n,1≤t≤2n 的所有正整数 s,ts,ts,t ，你需要计算如果由由子从 sss /th座位开始，移动 ttt 秒，她可以吃掉的菜肴的最大总份量。
